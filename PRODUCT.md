# Product

## Register

brand

## Users

三类受众，按层级阅读：
- **HR / 技术招聘者**（主）：30秒内判断是否值得深聊，需要快速抓取核心标签（技术栈、工作年限、领域亮点）
- **技术面试官 / 工程师**（次）：愿意深读，关注技术深度、项目细节、代码品位
- **潜在合作伙伴 / 客户**（辅）：关注能解决什么问题，以成果和案例为判断依据

访客打开页面时，通常坐在办公室的电脑前做候选人筛选，或在咖啡馆深夜浏览感兴趣的开发者主页。

## Product Purpose

高级前端工程师辞鸢（Yonjay Doyle）的个人作品集 + 技术博客。
核心目的：**让访客在离开前形成"这个人真的在认真做事"的具体印象**，而非泛泛"技术不错"。

成功标准：访客能在两分钟内准确复述出三件关于这个人的具体事情（不是"React做得好"，而是"他用vtk.js做了肺部三维重建"）。

## Brand Personality

沉浸 · 精工 · 可信

情感目标：访客应感受到一种"这个人不一样，而且我可以信任这种不一样"——不是酷，是有说服力的独特。

## Anti-references

- Bootstrap 风格简历模板（时间轴 + 技能进度条）
- SaaS Landing Page 营销套路（大标题 + Feature cards + CTA 堆砌）
- 暗色极客风（Matrix 绿字 / 赛博朋克霓虹 / 终端字体堆砌）
- 看一眼就知道是"套用 shadcn 预设"的雷同感

## Design Principles

1. **场景即叙事**：3D/动效不是装饰，是讲故事的媒介——每一帧都应有意图，不存在"只是好看"
2. **精度即可信**：每个像素级决策（间距、字重、颜色）都是工程师思维的投影，随意感等于失去信任
3. **密度服从层次**：信息分三层呈现——表层（HR 扫视）→ 中层（工程师停留）→ 深层（合作伙伴细读）；不同访客都能在自己的层找到满足感
4. **克制中的意外**：整体克制，但在关键节点（首屏进入、项目展开）保留一个真正令人意外的视觉时刻
5. **实践即展示**：作品集本身的技术实现（Three.js 场景、动效细节、性能表现）就是能力的直接证明

## Accessibility & Inclusion

WCAG AA 基线（颜色对比、键盘可访问性），视觉效果优先，不做超出基线的特殊适配。
提供 reduced-motion 支持，降级后内容完整不受影响。
